# Shinkansen Live

Real-time animated map of Japan's Shinkansen bullet train network. Trains move along their actual routes in JST, updated every second.

## Features

- **Live train map** — Animated markers interpolated along real track geometry from OpenStreetMap
- **Timetable viewer** — All 8 Shinkansen lines, English/Japanese toggle, horizontally scrollable tables
- **Live & Playback modes** — Watch trains in real time or scrub to any time of day
- **9 map layers** — Google Maps (Road/Satellite/Hybrid/Terrain), OpenStreetMap, OpenRailwayMap, CartoDB (Dark Matter/Positron/Voyager)
- **Live cameras** — Embedded YouTube streams of Shinkansen stations
- **Line toggle** — Show/hide individual lines and station labels
- **Planned routes** — Chuo Shinkansen, Hokkaido extension, and others shown as dashed lines

## Quick Start

```bash
git clone https://github.com/manavya-des/shinkansen-live-open.git
cd shinkansen-live-open
npm install
npm run dev
```

Open [localhost:3000](http://localhost:3000). The app ships with pre-built timetable and animation data, so it works immediately.

## Customizing Data

### Timetable

Timetable data lives in [`timetable/data.json`](timetable/data.json). See [`timetable/README.md`](timetable/README.md) for the full schema reference.

### Live Cameras

Live camera feeds are configured in [`cameras/data.json`](cameras/data.json). See [`cameras/README.md`](cameras/README.md) for field documentation. Add YouTube live stream URLs to show camera markers on the map.

### Rebuilding

After editing either file:

```bash
npm run build:data    # Regenerates animation data + copies timetable & cameras into src/data/
```

## Data Pipeline

### 1. Track Geometry

```bash
node scripts/fetch-all-shinkansen.mjs
```

Queries OpenStreetMap's Overpass API for Shinkansen rail relations. Orders track segments into continuous paths between terminal stations. Outputs GeoJSON-like features with `[lng, lat]` coordinates into `src/data/shinkansen-lines.json`.

### 2. Timetable

Edit `timetable/data.json` directly. The format is a flat list of lines, each with two directions (downbound/upbound), containing all trains and station times. See [`timetable/README.md`](timetable/README.md) for field-by-field documentation.

### 3. Animation Data

```bash
npm run build:data
```

Reads `timetable/data.json` and station coordinates from the map component. For each route:
- Combines Tokaido + Sanyo + Kyushu into one western route
- Deduplicates trains that appear across overlapping lines
- Converts `HH:MM` times to minute-of-day values
- Interpolates pass-through times based on haversine distance ratios

Outputs `src/data/shinkansen-train-animations.json` and copies the timetable to `src/data/shinkansen-timetables.json`.

## How Train Animation Works

1. **Route templates** are built at init: track polyline coordinates are projected and measured, station positions are snapped to the path with cumulative distances
2. **Every second**, the current JST time is converted to minutes-of-day
3. For each active train, the animation finds which segment it's on (between two timed stations) and linearly interpolates the position along the path
4. Train markers (`CircleMarker`) are created/updated/removed from the Leaflet layer
5. Trains inherit the color of whichever line segment they're currently on (e.g. a Nozomi turns from Tokaido orange to Sanyo blue at Shin-Osaka)
